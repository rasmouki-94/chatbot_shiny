PROBLÈME 1 — Barre de progression (dynamique + %)
OBJECTIF UX
- Barre très épurée, fine, arrondie, style Apple.
- Accent color #8e66c7.
- Elle doit avancer AU FUR ET À MESURE du parcours, avec un pourcentage affiché de façon discrète.
- Transition douce (250–400ms). Aucun effet flashy.
- Doit fonctionner avec le rendu “chat” (message queue) sans dépendre d’un rerender complet de l’UI.

EXIGENCES TECHNIQUES
- Créer une source de vérité unique `progress` (0–100) calculée dynamiquement selon l’étape :
  - Questions : progression linéaire de 0% à 70% sur N questions (N = nb de questions effectif)
  - Lead gate (prénom/nom/entreprise/email) : 70% → 85% (linéaire sur 4 étapes)
  - Résultat : passer à 100% quand la carte résultat est affichée
- À chaque avancée (réponse question, étape lead gate validée, résultat affiché), appeler une fonction unique `setProgress(pct)` qui :
  - met à jour la largeur du “fill” de la barre
  - met à jour le texte “XX%” (discret, petit)
- IMPORTANT : la mise à jour doit se faire côté front via custom messages (Shiny custom message handler) pour éviter les bugs liés aux rerenders Shiny.
  - Exemple attendu :
    - server : session$sendCustomMessage("progress", list(pct = pct))
    - JS : Shiny.addCustomMessageHandler("progress", function(msg){ ... })
- Si mon UI est en React, faire l’équivalent via un event / state update ; sinon, JS vanilla.

À FAIRE
1) Diagnostiquer pourquoi la barre ne bouge pas (IDs DOM, rerender, CSS width, handler absent, conflit).
2) Corriger avec la stratégie ci-dessus (custom messages + handler robuste).
3) S’assurer que la barre est visible et ne “saute” pas.
4) Afficher le pourcentage : ex “42%” aligné à droite du header, en gris discret.
